Travail semaine 2:

Il est consacré à l’apprentissage de HTML et CSS.


Cette semaine, je commence à programmer : super ! En lisant le programme, je me 
rends compte qu’il n’y a aucun exercice à faire, qu’il n’y a pas non plus d’exercices 
corrigés à prendre en exemple => Panique !!! J’écris un message sur le forum pour
faire part de mon désarroi et de mon incapacité à entrer une ligne de code dans quoi
que ce soit. C’est là qu’un stagiaire de la même formation que moi, Henri, m’envoie
un mail pour me dire que je ne suis pas seule dans cette situation, et qu’il ne faut
surtout pas que je perde courage et m’explique comment il commence. N’est-ce pas que
c’est formidable le web !!!, et là commence un échange de mails pour essayer d’avancer
l’un et l’autre, et aussi envoi de questions sur le forum pour essayer de progresser 
au mieux.

Ce que je fais :
Comme la semaine dernière, je lis le programme donné pour la semaine. Ensuite, je 
décide de démarrer par HTML et de m’intéresser ensuite à CSS.

Je commence par créer un repository semaine2 dans github dans lequel je viendrais 
ranger les programmes que je vais écrire cette semaine.
Pour programmer, l’éditeur à utiliser est cloud9. Il faut créer un compte, ouvrir
le  workspace associé au repository créé juste avant. En ce qui me concerne, j’ai
cliqué sur HTML quand il me demande de choisir un langage. 
Un écran apparaît , avant d’aller plus loin, cliquer sur ce lien pour apprendre à
créer un fichier et à le sauvegarder.

Exemple de création et de sauvegarde de fichier avec cloud9 

Faire la même chose sous cloud9 . Commencer à taper du code (un petit programme 
tout simple, celui donné en exemple dans la doc C9), cliquer sur Preview en haut 
de l’écran (à côté de run) puis sur Live Preview File, une fenêtre va s’ouvrir qui
visualise le résultat du codage. C’est assez formidable puisqu’ainsi, je vois en 
temps réel ce que je programme. 
Je sauvegarde le code que j’écris en appuyant régulièrement sur les touches
Ctrl et S.

Dans le bas de l’écran, il y a une fenêtre bash, c’est dans celle-ci que je vais 
entrer les commandes git vues la semaine dernière. Je retourne faire un tour dans
le cours gérer son code avec git et github pour me les remettre à l'esprit.

A ce stade, je fais un git add nomfichier.html pour que mon fichier puisse être
« commitable ».
Si je fais un git status, je vois mon fichier inscrit en rouge, ça veut dire qu’il
n’est pas commité.
Je tape git commit –m «  petit programme en html » , puis git status, mon fichier
apparaît en vert. J’ai donc fait une photographie qui comprend uniquement mon 
petit programme, j’aurais pu faire un commit après avoir créé plusieurs programmes,
ça aurait très bien fonctionné.
A la fin de ma journée, après avoir fait plusieurs petits programmes, plusieurs 
commits, je décide de faire un git push. Pour le moment, mes programmes sont connus
sous cloud9, pas dans mon repository de github, le push sert à envoyer mes
programmes dans ce repository et à les rendre publics.
Je retourne ensuite sur Github, je retrouve bien dans mon repository semaine 2 mes 
petits programmes.

Ensuite, je passe pas mal de temps dans le site W3schools, il est très bien fait, il 
permet de prendre connaissance du langage HTML avec des exemples simples.

Je termine la semaine en regardant CSS, en faisant quelques essais à l'intérieur 
d'un programme html puis en créant un fichier style.css auquel un programme html
fait référence.
